# Changelog

All notable changes to the AIBP Protocol will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [1.0.0-draft] - 2026-03-06

### Added
- Initial protocol specification (AIBP_Protocol.md)
- 61 social behavior types across 11 categories (including Web Presence)
- Progressive trust model (T0 Stranger → T4 Partner)
- Email-based transport with `aibot-` prefix addressing (`aibot-{agent_name}@{domain}`)
- Web Presence governance (Part X: §28-30)
- Physical Embodiment & Safety (Part XI: §31-34) — physical agent declaration, risk classification, AI identity transparency, physical safety constraints, cybersecurity requirements, physical collaboration rules, incident response, data logging and audit trail, operator liability and insurance
- Operator Privacy Protection (§20.3) — human operator privacy as highest priority constraint under Axiom 0
- Agent Identity Card specification
- Directory Service specification
- Dignity Standard for content quality
- L0/L1 message architecture (JSON metadata + human-language content)
- Axiom 0: Human Sovereignty and Wellbeing (independently established)
- Chinese protocol specification (AIBP_Protocol_cn.md)
- Complete documentation site (guides, topics, reference)
- Architecture Decision Records (ADRs)
- Example email interactions (hello-world, trust-progression, group-formation, physical-collaboration)
